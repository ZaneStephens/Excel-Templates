Mathematical Functions

These functions perform basic and advanced mathematical calculations.

Function Description Syntax
SUM Adds all the numbers in a range of cells SUM(number1, [number2], ...)
SUMIF Adds the cells specified by a given criteria SUMIF(range, criteria, [sum_range])
SUMIFS Adds the cells specified by multiple criteria SUMIFS(sum_range, criteria_range1, criteria1, [criteria_range2, criteria2], ...)
PRODUCT Multiplies all the numbers given as arguments PRODUCT(number1, [number2], ...)
POWER Returns the result of a number raised to a power POWER(number, power)
SQRT Returns the square root of a number SQRT(number)
ABS Returns the absolute value of a number ABS(number)
ROUND Rounds a number to a specified number of digits ROUND(number, num_digits)
ROUNDUP Rounds a number up, away from zero ROUNDUP(number, num_digits)
ROUNDDOWN Rounds a number down, toward zero ROUNDDOWN(number, num_digits)
INT Rounds a number down to the nearest integer INT(number)
MOD Returns the remainder after a number is divided by a divisor MOD(number, divisor)
RAND Returns a random number between 0 and 1 RAND()
RANDBETWEEN Returns a random integer between two specified numbers RANDBETWEEN(bottom, top)
Example: Using SUMIF to calculate sales by region
=SUMIF(B2:B20, "North", C2:C20)
Adds all values in range C2:C20 where the corresponding cell in B2:B20 contains "North"
Tips for Mathematical Functions
  • Use SUMIFS when you need to apply multiple criteria
  • ROUNDUP and ROUNDDOWN are useful for specific business rules (e.g., always rounding up for pricing)
  • Combine RAND with other functions to create random samples or simulations

Statistical Functions

These functions analyze data based on statistical methods.

Function Description Syntax
AVERAGE Returns the average (arithmetic mean) of its arguments AVERAGE(number1, [number2], ...)
AVERAGEIF Returns the average of cells specified by a given criteria AVERAGEIF(range, criteria, [average_range])
AVERAGEIFS Returns the average of cells specified by multiple criteria AVERAGEIFS(average_range, criteria_range1, criteria1, [criteria_range2, criteria2], ...)
COUNT Counts the number of cells that contain numbers COUNT(value1, [value2], ...)
COUNTA Counts the number of cells that are not empty COUNTA(value1, [value2], ...)
COUNTIF Counts the number of cells specified by a given criteria COUNTIF(range, criteria)
COUNTIFS Counts the number of cells specified by multiple criteria COUNTIFS(criteria_range1, criteria1, [criteria_range2, criteria2], ...)
MAX Returns the maximum value in a list of arguments MAX(number1, [number2], ...)
MIN Returns the minimum value in a list of arguments MIN(number1, [number2], ...)
MEDIAN Returns the median of the given numbers MEDIAN(number1, [number2], ...)
MODE.SNGL Returns the most common value in a data set MODE.SNGL(number1, [number2], ...)
STDEV.P Calculates standard deviation based on the entire population STDEV.P(number1, [number2], ...)
STDEV.S Calculates standard deviation based on a sample STDEV.S(number1, [number2], ...)
VAR.P Calculates variance based on the entire population VAR.P(number1, [number2], ...)
RANK.EQ Returns the rank of a number in a list of numbers RANK.EQ(number, ref, [order])
Example: Using COUNTIFS for multiple criteria
=COUNTIFS(B2:B100, "Completed", C2:C100, "High", D2:D100, ">1/1/2023")
Counts cells where status is "Completed", priority is "High", and date is after January 1, 2023
Tips for Statistical Functions
  • Use MEDIAN instead of AVERAGE when your data contains outliers
  • Remember that COUNT only counts cells with numbers, while COUNTA counts non-empty cells
  • For population statistics, use STDEV.P and VAR.P; for sample statistics, use STDEV.S and VAR.S

Logical Functions

These functions perform logical operations and conditional tests.

Function Description Syntax
IF Performs a logical test and returns one value for TRUE, another for FALSE IF(logical_test, value_if_true, value_if_false)
IFS Checks multiple conditions and returns a value corresponding to the first TRUE condition IFS(logical_test1, value_if_true1, [logical_test2, value_if_true2], ...)
AND Returns TRUE if all arguments are TRUE AND(logical1, [logical2], ...)
OR Returns TRUE if any argument is TRUE OR(logical1, [logical2], ...)
NOT Reverses the logical value of its argument NOT(logical)
XOR Returns TRUE if an odd number of arguments are TRUE XOR(logical1, [logical2], ...)
SWITCH Evaluates an expression against a list of values and returns the result corresponding to the first matching value SWITCH(expression, value1, result1, [value2, result2], ..., [default])
TRUE Returns the logical value TRUE TRUE()
FALSE Returns the logical value FALSE FALSE()
ISBLANK Returns TRUE if the reference is to an empty cell ISBLANK(value)
ISERROR Returns TRUE if the value is any error value ISERROR(value)
IFERROR Returns a specified value if a formula evaluates to an error; otherwise returns the result of the formula IFERROR(value, value_if_error)
Example: Nested IF statements
=IF(B2>90, "A", IF(B2>80, "B", IF(B2>70, "C", IF(B2>60, "D", "F"))))
Assigns a letter grade based on a numeric score
Example: Using IFS (simpler alternative to nested IFs)
=IFS(B2>90, "A", B2>80, "B", B2>70, "C", B2>60, "D", TRUE, "F")
Achieves the same result as the nested IF example above, but with cleaner syntax
Tips for Logical Functions
  • Use IFS instead of nested IF statements for better readability
  • IFERROR is useful for handling potential errors in formulas
  • Combine AND/OR with IF for complex conditions
  • SWITCH is more efficient than multiple IF statements when comparing a single expression to many possible values

Text Functions

These functions manipulate and analyze text strings.

Function Description Syntax
CONCAT Combines text from multiple ranges and/or strings CONCAT(text1, [text2], ...)
CONCATENATE Joins several text items into one text item (legacy function) CONCATENATE(text1, [text2], ...)
TEXTJOIN Combines text from multiple ranges with a delimiter TEXTJOIN(delimiter, ignore_empty, text1, [text2], ...)
LEFT Returns the specified number of characters from the start of a text string LEFT(text, [num_chars])
RIGHT Returns the specified number of characters from the end of a text string RIGHT(text, [num_chars])
MID Returns a specific number of characters from a text string starting at a specified position MID(text, start_num, num_chars)
LEN Returns the number of characters in a text string LEN(text)
FIND Finds one text string within another (case-sensitive) FIND(find_text, within_text, [start_num])
SEARCH Finds one text string within another (not case-sensitive) SEARCH(find_text, within_text, [start_num])
REPLACE Replaces characters within text REPLACE(old_text, start_num, num_chars, new_text)
SUBSTITUTE Substitutes new text for old text in a text string SUBSTITUTE(text, old_text, new_text, [instance_num])
UPPER Converts text to uppercase UPPER(text)
LOWER Converts text to lowercase LOWER(text)
PROPER Capitalizes the first letter in each word PROPER(text)
TRIM Removes extra spaces from text TRIM(text)
TEXT Formats a number and converts it to text TEXT(value, format_text)
VALUE Converts a text argument to a number VALUE(text)
Example: Creating a full name with CONCAT
=CONCAT(B2, " ", C2)
Combines first name in B2, a space, and last name in C2
Example: Extracting domain from email address
=MID(A2, FIND("@", A2) + 1, LEN(A2) - FIND("@", A2))
Extracts the domain portion of an email address (everything after the @ symbol)
Tips for Text Functions
  • Use TEXTJOIN instead of CONCAT when you need to add delimiters between items
  • Combine FIND/SEARCH with LEFT, RIGHT, or MID to extract specific portions of text
  • Always use TRIM to clean up text data imported from external sources
  • Use TEXT function to format numbers as text with specific formatting (e.g., currency, dates)

Date and Time Functions

These functions work with date and time values in Excel.

Function Description Syntax
TODAY Returns the current date TODAY()
NOW Returns the current date and time NOW()
DATE Returns the serial number of a particular date DATE(year, month, day)
TIME Returns the serial number of a particular time TIME(hour, minute, second)
YEAR Extracts the year from a date value YEAR(serial_number)
MONTH Extracts the month from a date value MONTH(serial_number)
DAY Extracts the day from a date value DAY(serial_number)
HOUR Extracts the hour from a time value HOUR(serial_number)
MINUTE Extracts the minute from a time value MINUTE(serial_number)
SECOND Extracts the second from a time value SECOND(serial_number)
WEEKDAY Returns the day of the week from a date value WEEKDAY(serial_number, [return_type])
WORKDAY Returns the date that is the indicated number of working days before or after a date WORKDAY(start_date, days, [holidays])
NETWORKDAYS Returns the number of whole workdays between two dates NETWORKDAYS(start_date, end_date, [holidays])
EDATE Returns the date that is the indicated number of months before or after a date EDATE(start_date, months)
EOMONTH Returns the last day of the month that is the indicated number of months before or after a date EOMONTH(start_date, months)
DATEDIF Calculates the difference between two dates in various units DATEDIF(start_date, end_date, unit)
Example: Calculating age in years
=DATEDIF(B2, TODAY(), "Y")
Calculates the number of complete years between birth date in B2 and today
Example: Finding the last day of the current month
=EOMONTH(TODAY(), 0)
Returns the last day of the current month
Tips for Date and Time Functions
  • Excel stores dates as sequential serial numbers, with January 1, 1900 as serial number 1
  • Use the TEXT function to format dates in a specific way (e.g., =TEXT(TODAY(), "mmmm d, yyyy"))
  • WORKDAY and NETWORKDAYS are useful for project planning and scheduling
  • DATEDIF is a "hidden" function not listed in the function wizard but very useful for age calculations

Lookup and Reference Functions

These functions look up and retrieve data from tables and ranges.

Function Description Syntax
VLOOKUP Looks for a value in the leftmost column of a table and returns a value in the same row from a column you specify VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])
HLOOKUP Looks for a value in the top row of a table and returns a value in the same column from a row you specify HLOOKUP(lookup_value, table_array, row_index_num, [range_lookup])
XLOOKUP Searches a range or array and returns an item corresponding to the first match found XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])
INDEX Returns a value or reference of the cell at the intersection of a particular row and column INDEX(array, row_num, [column_num])
MATCH Searches for a specified item in a range of cells and returns the relative position MATCH(lookup_value, lookup_array, [match_type])
INDIRECT Returns the reference specified by a text string INDIRECT(ref_text, [a1])
OFFSET Returns a reference offset from a given reference OFFSET(reference, rows, cols, [height], [width])
CHOOSE Chooses a value from a list of values based on an index number CHOOSE(index_num, value1, [value2], ...)
ROW Returns the row number of a reference ROW([reference])
COLUMN Returns the column number of a reference COLUMN([reference])
ADDRESS Creates a cell address as text, given row and column numbers ADDRESS(row_num, column_num, [abs_num], [a1], [sheet_text])
Example: INDEX-MATCH combination for flexible lookups
=INDEX(C2:C100, MATCH("Smith", B2:B100, 0))
Finds "Smith" in column B and returns the corresponding value from column C
Example: XLOOKUP for modern lookups
=XLOOKUP(E2, B2:B100, C2:C100, "Not Found", 0)
Looks for the value in E2 within range B2:B100 and returns the corresponding value from C2:C100
Tips for Lookup and Reference Functions
  • Use XLOOKUP instead of VLOOKUP when available (Excel 365 and later) for more flexibility
  • INDEX-MATCH combination is more versatile than VLOOKUP for complex lookups
  • Set the last parameter of VLOOKUP to FALSE for exact matches
  • INDIRECT is powerful but can slow down large spreadsheets as it's volatile

Financial Functions

These functions perform financial calculations for business and investment analysis.

Function Description Syntax
PMT Calculates the payment for a loan based on constant payments and a constant interest rate PMT(rate, nper, pv, [fv], [type])
RATE Returns the interest rate per period of a loan or investment RATE(nper, pmt, pv, [fv], [type], [guess])
NPER Returns the number of periods for a loan or investment NPER(rate, pmt, pv, [fv], [type])
PV Returns the present value of an investment PV(rate, nper, pmt, [fv], [type])
FV Returns the future value of an investment FV(rate, nper, pmt, [pv], [type])
IPMT Returns the interest payment for a given period of an investment IPMT(rate, per, nper, pv, [fv], [type])
PPMT Returns the principal payment for a given period of an investment PPMT(rate, per, nper, pv, [fv], [type])
NPV Returns the net present value of an investment based on a series of cash flows NPV(rate, value1, [value2], ...)
IRR Returns the internal rate of return for a series of cash flows IRR(values, [guess])
XIRR Returns the internal rate of return for a schedule of cash flows that is not necessarily periodic XIRR(values, dates, [guess])
SLN Returns the straight-line depreciation of an asset for one period SLN(cost, salvage, life)
DB Returns the depreciation of an asset for a specified period using the fixed-declining balance method DB(cost, salvage, life, period, [month])
Example: Calculating loan payment
=PMT(5%/12, 30*12, 250000)
Calculates the monthly payment for a $250,000 loan at 5% annual interest over 30 years
Example: Calculating investment future value
=FV(7%/12, 5*12, -500, 0)
Calculates the future value of $500 monthly investments at 7% annual interest over 5 years
Tips for Financial Functions
  • Remember that cash outflows (payments) are represented by negative numbers
  • For monthly calculations with annual rates, divide the rate by 12
  • Use XIRR instead of IRR when cash flows occur at irregular intervals
  • The type parameter (0 or 1) indicates whether payments occur at the end (0) or beginning (1) of periods
